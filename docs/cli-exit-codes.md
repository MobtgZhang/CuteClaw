# CLI 退出码

| 码 | 含义 | 典型场景 |
|----|------|----------|
| 0 | 成功 | 正常执行完毕 |
| 1 | 校验或业务拒绝 | `validate` 未通过；`config validate` 的 `schema_version` 不支持等 |
| 2 | 用法错误、I/O 或解析失败 | 缺参数；快照/config 不存在且无法继续；`import` JSON 无法解析；`status` 无法加载 store |

未单独列出的子命令若通过 `try` 传播 Zig 错误，可能由运行时打印栈迹并以非 0 退出（多用于缺陷而非用户误操作）。
